Quick reference

Page opening

Home page / marketing landing page #hero
Interior / department / service page #page-header

Mid-page patterns

Introduce a section #section-intro
3–6 parallel items / features #card-group
Image + text, side by side #media-feature
Image on the right side #media-feature --right
Statistics / key numbers #factoid-collection
FAQ / collapsible items #accordion
Single spotlight or CTA #callout
System notice or status #alert
List of links / navigation #link-list

Components

Hero #hero

Full-bleed image hero for home pages and marketing landing pages. Always the first element on the page. Never used alongside #page-header.

Use for

  • home page
  • homepage
  • landing page
  • campaign page
  • marketing page
  • splash page
  • full-bleed image
  • big image banner
  • opening banner
  • site intro

Avoid when

  • Interior or department page — use #page-header
  • Page already has a #page-header
  • No meaningful image to support the hero

Page Header #page-header

Primary page introduction for standard interior pages. Replaces the hero on non-home pages. Supports an optional image and CTA button.

Use for

  • interior page
  • department page
  • service page
  • topic page
  • section page
  • about page
  • standard page intro
  • page title and description

Avoid when

  • Home page or marketing landing — use #hero
  • Page already has a #hero
  • Mid-page section intro — use #section-intro

Section Intro #section-intro

A lightweight heading + description used to introduce a content section mid-page. Supports background variants for on-dark contexts.

Use for

  • section heading
  • section title
  • group intro
  • before cards
  • before accordion
  • before factoids
  • mid-page intro
  • content section header

Avoid when

  • Full page-level intro — use #page-header or #hero
  • Needs a CTA button — use #callout instead

Background variants

--maroonMaroon (on dark) — #500000, white text

Card Group #card-group

A responsive grid of 3–6 equal-weight cards, each with heading, body text, optional icon, and link. Columns auto-size based on card count via data-cards.

Use for

  • cards
  • card grid
  • features
  • services
  • resources
  • parallel items
  • topic grid
  • collection overview
  • 3 items
  • 4 items
  • 5 items
  • 6 items

Avoid when

  • Fewer than 3 items — use #callout
  • Items have clear hierarchy — use #link-list
  • Only 1–2 items

Background variants

FlagBackground
noneWhite (default)
--gray-100Light gray
--maroonMaroon (on dark)
--gray-900Charcoal (on dark)
Syntax note: Use ## Heading per card. Add icon: name and link: url below each heading as needed.
Media Feature #media-feature

Side-by-side image and text layout. Image on the left by default; add --right to flip. Alternate orientations when using multiple media features in a row.

Use for

  • image with text
  • split layout
  • feature section
  • photo and description
  • image left
  • image right
  • side by side
  • topic spotlight
  • alternating sections

Avoid when

  • Primary hero needed — use #hero
  • Standard interior page intro — use #page-header
  • No image available

Factoid Collection #factoid-collection

A grid of statistics or key numbers with labels. Use to create visual impact with quantitative data. Supports an optional superhead and title above the grid.

Use for

  • statistics
  • stats
  • numbers
  • data points
  • key figures
  • by the numbers
  • impact numbers
  • annual report
  • metrics

Avoid when

  • Fewer than 3 stats — use inline body copy
  • Numbers need detailed explanation — use #card-group

Background variants

FlagBackground
noneWhite (default)
--gray-100Light gray
--maroonMaroon (on dark)
--gray-900Charcoal (on dark)
Syntax: stat: value | label per item. Example: stat: 4.5 million | Physical volumes
Accordion #accordion

Collapsible FAQ or Q&A list rendered with native <details>/<summary> elements. Each ## Heading becomes one expandable item.

Use for

  • FAQ
  • frequently asked questions
  • Q&A
  • collapsible
  • expandable
  • show/hide
  • toggle
  • help topics
  • how-to list
  • policies

Avoid when

  • Content should always be visible — use body copy or cards
  • Fewer than 2 items — use a #callout

Callout #callout

A visually highlighted block spotlighting a single fact, feature, or action. Use to break up sections and draw attention to one key piece of content.

Use for

  • callout
  • highlight
  • spotlight
  • featured content
  • call to action
  • promotional block
  • visual break
  • fun fact
  • did you know
  • program highlight

Avoid when

  • System message or status — use #alert
  • Multiple items — use #card-group
  • Permanent editorial content — use body copy or #media-feature

Alert #alert

A contextual banner for time-sensitive system or status messages. The severity variant is required — choose one of the four type flags below.

Use for

  • alert
  • notice
  • warning
  • error
  • system message
  • status update
  • closure notice
  • hours change
  • service disruption
  • time-sensitive

Avoid when

  • Editorial or promotional content — use #callout
  • Permanent page content — use body copy or #callout

Severity variants (required)

FlagLabelDescriptionTrigger keywords
--info Info (blue) General informational message. Default when no severity applies.
infonoticefyiblue
--success Success (green) Positive confirmation or completed action.
successconfirmedcompletegreen
--warning Warning (amber) Non-critical caution or advisory.
warningcautionattentionamber
--error Error (red) Critical problem or failure state.
errorfailurecriticalurgentred
Note: Avoid rendering the alert icon on mobile without testing — the SVG sprite can render oversized if it hasn't loaded. When in doubt, omit the icon.
Link List #link-list

A vertical list of labeled links with optional descriptions. Use when items have a clear hierarchy or are primarily navigational rather than equal-weight content.

Use for

  • links
  • link list
  • navigation list
  • resource list
  • directory
  • index
  • related links
  • quick links
  • hierarchical items
  • table of contents

Avoid when

  • Equal-weight items with images — use #card-group
  • Expandable items — use #accordion

Background variants

FlagBackground
noneWhite (default)
--gray-100Light gray
--maroonMaroon
Syntax: Each ## Heading is a link label. Add link: url below each item. Optional body text becomes the item description.

Shared background variants

These four background flags are available on most components. Maroon and charcoal are "on-dark" contexts — the builder automatically applies inverted text colors.

Default White — #ffffff
--gray-100 Light gray — #f5f5f0
--maroon Maroon — #500000
--gray-900 Charcoal — #1c1b18
